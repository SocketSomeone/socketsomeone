# Projects Tech Stack Design

**Date:** 2026-04-19

**Goal**

Добавить на страницу `Projects` отдельную секцию с бенто-композицией, которая показывает основной стек технологий автора через иконки, подписи и спокойные премиальные анимации.

**Context**

Текущая страница `Projects` уже содержит:
- hero-заголовок секции;
- список проектов;
- блок `Wakatime`.

Новый блок должен:
- располагаться между списком проектов и `Wakatime`;
- быть вынесен в отдельный компонент, чтобы страница оставалась читаемой;
- сохранять визуальный язык проекта, но давать более выразительный technical showcase.

**User Decisions**

- Не разделять стек на `Languages`, `Software` и `Setup`.
- `Setup` не нужен.
- Нужен один общий блок со стеком.
- Визуальный стиль: плотный техничный бенто.
- Анимации: спокойные премиальные, без агрессивного движения.
- Архитектура: вся логика в компонентах, страница только подключает готовый блок.

**Proposed Section**

Новая секция `Tools I work with`:
- отдельный заголовок;
- короткое описание;
- одна крупная `bento`-карточка с внутренними зонами.

**Bento Composition**

Внутри секции используется одна внешняя оболочка и несколько смысловых зон:

1. `Core stack`
   Крупная зона с главным стеком:
   - TypeScript
   - Node.js
   - NestJS
   - React
   - Next.js

2. `Infrastructure`
   Средняя зона для платформенных инструментов:
   - Docker
   - Kubernetes
   - Nginx
   - GitHub Actions

3. `Data & Messaging`
   Средняя зона для баз данных и интеграционных технологий:
   - PostgreSQL
   - Redis
   - RabbitMQ
   - GraphQL

4. `Also worked with`
   Широкая нижняя зона для вторичного стека в виде компактных чипов.

**Visual Direction**

- Основа блока: rounded контейнер с мягким градиентным фоном.
- Поверхность карточек: border + полупрозрачная подложка.
- Главные технологии отображаются крупнее второстепенных.
- У всех иконок есть текстовые подписи.
- Блок должен хорошо работать в светлой и темной теме.

**Animation Direction**

Анимации через `motion`:
- секция плавно появляется при попадании во viewport;
- зоны появляются со stagger-анимацией;
- на hover карточки слегка поднимаются и усиливают glow;
- фон получает очень медленное, едва заметное движение;
- никаких бесконечных прыжков, сильного parallax или шумных эффектов.

**Architecture**

Страница `app/[locale]/projects/page.tsx` должна только рендерить готовую секцию.

Новый блок строится компонентно:
- основной компонент секции;
- переиспользуемый компонент карточки категории;
- локальная декларация данных стека;
- при необходимости отдельный компонент для логотипа технологии.

**Responsive Behavior**

- Desktop: выразительная бенто-композиция с несколькими зонами.
- Tablet: двухколоночная сетка с сохранением иерархии.
- Mobile: одна колонка, без потери читаемости и без перегрузки логотипами.

**Constraints**

- Не смешивать код новой секции с кодом списка проектов.
- Не добавлять громоздкую логику на саму страницу.
- Не превращать блок в простую таблицу логотипов.
- Не использовать слишком шумные анимации.

**Testing Expectations**

- Секция корректно рендерится на странице `Projects`.
- Нет ошибок гидратации.
- Layout остается читаемым на мобильных и десктопных ширинах.
- Анимации не ломают SSR/CSR поведение.
